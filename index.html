Selected Code Patterns for Vascular Research

Working snippets I have used or adapted. Each is reproducible against a registry-shaped data frame and is illustrative of how I structure analyses for vascular outcomes work.

Riptw_vascular_outcomes.R
library(WeightIt)
library(cobalt)
library(survey)

# Inverse probability of treatment weighting for open vs endovascular
wt <- weightit(
  treatment ~ age + sex + diabetes + ckd + smoking + asa_class + baseline_abi,
  data       = vascular_df,
  method     = "ps",
  estimand   = "ATE",
  stabilize  = TRUE
)

vascular_df$w <- wt$weights
love.plot(wt, thresholds = c(m = 0.1), abs = TRUE)

des <- svydesign(ids = ~1, weights = ~w, data = vascular_df)
fit <- svyglm(major_complication_30d ~ treatment,
              design = des, family = quasibinomial())
summary(fit)
exp(cbind(OR = coef(fit), confint(fit)))
Use case: risk-adjusted comparison of open versus endovascular procedures while preserving cohort size and producing reviewer-ready balance diagnostics.
Ramputation_free_survival.R
library(survival)
library(survminer)

# Amputation-free survival for CLTI revascularization
surv_obj <- Surv(time  = clti_df$followup_days,
                 event = clti_df$major_amputation_or_death)

cox_fit <- coxph(
  surv_obj ~ revascularization + age + diabetes + renal_failure + wound_grade,
  data = clti_df
)

cox.zph(cox_fit)  # proportional hazards check

ggsurvplot(survfit(cox_fit), data = clti_df,
           risk.table = TRUE, conf.int = TRUE,
           legend.title = "Strategy")
Use case: CLTI, dialysis access, graft patency, EVAR / TEVAR reintervention, or restenosis endpoints, with a documented PH assumption check.
Rcompeting_risks_reintervention.R
library(cmprsk)

# Fine-Gray subdistribution hazard for freedom from reintervention,
# with death as a competing risk
fg <- crr(
  ftime    = evar_df$followup_days,
  fstatus  = evar_df$event_code,   # 0 censored, 1 reintervention, 2 death
  cov1     = model.matrix(~ age + sex + aaa_diameter + endoleak_type,
                          data = evar_df)[, -1],
  failcode = 1,
  cencode  = 0
)
summary(fg)
Use case: when mortality precludes the event of interest (a typical situation in elderly AAA cohorts), competing-risks modeling is the honest default.
Rmeta_analysis_vascular.R
library(meta)

# Random-effects meta-analysis with Hartung-Knapp adjustment
m <- metabin(
  event.e = events_drug,   n.e = total_drug,
  event.c = events_ctrl,   n.c = total_ctrl,
  studlab = trial,
  data    = trials_df,
  sm      = "RR",
  method.tau = "REML",
  hakn    = TRUE,
  prediction = TRUE
)
summary(m)
forest(m, leftcols = c("studlab", "event.e", "n.e", "event.c", "n.c"))
Use case: standard pooled estimate with a prediction interval, the way reviewers increasingly expect random-effects meta-analyses to be reported.
Pythonnsqip_cohort_pull.py
import duckdb, pandas as pd

con = duckdb.connect("nsqip_2017_2022.duckdb")

cohort = con.execute("""
  SELECT case_id, age, sex, race, asa_class,
         diabetes, smoker, ckd_stage,
         cpt_code, procedure_group,
         death_30d, readmission_30d, sssi_30d, length_of_stay
  FROM nsqip
  WHERE procedure_group IN ('EVAR','OPEN_AAA','CEA','CAS','LE_BYPASS')
    AND age BETWEEN 40 AND 90
""").df()

cohort["composite_morbidity"] = (
    cohort[["death_30d","readmission_30d","sssi_30d"]].max(axis=1)
)
cohort.to_parquet("vascular_cohort.parquet")
Use case: fast cohort extraction from a NSQIP-style dataset stored as DuckDB, ready for downstream PSM / IPTW analysis in R.

Workflow, Reproducibility, and Mentor Collaboration

A research fellowship is also a project-management problem. The workflow below is how I keep multiple projects moving, keep mentors unblocked, and keep manuscripts audit-ready months after submission.

Project Skeleton

Every project is a Git repository with a fixed layout: data/ (read-only), R/ or src/, analysis/, figures/, manuscript/, and a README.md with the question, primary outcome, and analysis plan. Environments pinned with renv for R and uv or conda for Python.

GitrenvuvREADME-driven

Variable Dictionary First

Before any modeling, a variable dictionary is drafted: name, source field, type, allowed values, derivation rules, and clinical justification. This is the single artifact biostatisticians, mentors, and reviewers ask for. Building it first prevents most downstream rework.

Data DictionarySTROBE

Pre-Specified Analysis Plan

For observational projects, the analysis plan is written before unblinding to outcomes: primary and secondary endpoints, covariate set, modeling approach, sensitivity analyses, and decisions on missingness. Deviations are documented, not hidden.

Pre-registrationSensitivity

Mentor Communication

Weekly written progress note: what shipped, what is blocked, what I need from the mentor by name. Calendar invites for any analytic decision that crosses a clinical judgment threshold. No surprise blockers at the end of the fellowship.

Weekly NoteDecisions Log

Figure Discipline

Figures are generated by code, not by hand. Every figure in a manuscript ties back to a versioned script and a captured data snapshot. Reviewer-requested re-runs are a single command, not an archaeology project.

ggplot2matplotlibQuarto
